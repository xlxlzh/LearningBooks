# 第 5 章 渲染管线

---

## 5.1 3D 错觉

在 2D 屏幕上呈现 3D 场景的关键，是利用人类视觉系统的几个基本特征来营造深度和体积的错觉。

**线性透视（Linear Perspective）**

平行线向远方延伸时，在人眼中会逐渐汇聚到一个消失点。例如铁轨的两条平行轨道，越远处看起来越靠近，最终在地平线处汇合。这是透视投影的基础原理。

**近大远小**

物体距离观察者越远，其在视网膜上的成像越小。同一尺寸的物体，近处的看起来大，远处的看起来小。这种大小随深度递减的现象是深度感知的重要线索。

**物体遮挡**

不透明的近处物体会遮挡远处的物体。这种遮挡关系直接传达了场景中的深度顺序——被遮挡的物体一定在遮挡者之后。

**光照与阴影**

光照决定了物体表面的明暗分布，揭示了物体的立体形状。一个球体在没有光照时看起来只是一个平面圆盘，而有了光照后就能清晰看出其球形体积。阴影则暗示了光源位置和物体之间的高度关系。

## 5.2 模型表示

3D 实体对象通过**三角形网格（Triangle Mesh）**来近似表示。三角形是构成 3D 模型的基本图元：

- 三个顶点确定一个平面，天然适合表示平面表面。
- 三角形在投影变换后仍然是三角形（凸多边形性质）。
- GPU 硬件专门针对三角形处理做了高度优化。

三角形的数量越多，模型细节越丰富，但处理开销也越大。实际开发中，建模师使用 3D Studio Max、Maya、Blender 等专业工具创建复杂模型，程序通过加载这些模型文件来渲染。

## 5.3 基本计算机颜色

显示器通过每个像素发射红、绿、蓝三原色的混合光来呈现颜色。通过控制每种颜色的强度，可以组合出数百万种颜色。

### 5.3.1 颜色运算

颜色向量 `(r, g, b)` 的各分量归一化到 `[0, 1]` 范围。支持以下运算：

- **加法**：`(0.0, 0.5, 0.0) + (0.0, 0.0, 0.25) = (0.0, 0.5, 0.25)` —— 混合两种颜色
- **标量乘法**：`0.5 × (1, 1, 1) = (0.5, 0.5, 0.5)` —— 降低亮度
- **调制（Modulation）**：`(r₁, g₁, b₁) ⊗ (r₂, g₂, b₂) = (r₁r₂, g₁g₂, b₁b₂)` —— 分量乘法，用于光照计算

运算结果超出 `[0, 1]` 范围时需要截断（clamp）。

### 5.3.2 128 位颜色

使用 4 个 32 位浮点数表示 `(r, g, b, a)`，其中 `a` 是 Alpha（透明度）通道。128 位颜色精度高，适合在像素着色器内部进行颜色计算。在代码中用 `XMVECTOR` 表示，可利用 SIMD 加速。

```cpp
// 颜色调制
XMVECTOR XM_CALLCONV XMColorModulate(FXMVECTOR C1, FXMVECTOR C2);
```

### 5.3.3 32 位颜色

使用 4 个 8 位无符号整数表示颜色，每个分量范围 `[0, 255]`，总共可表示 `256³ = 16,777,216` 种颜色。32 位颜色存储紧凑，适合作为最终像素值写入后台缓冲区。

```cpp
// DirectXMath 提供的 32 位颜色结构
namespace DirectX::PackedVector {
    struct XMCOLOR {
        union {
            struct { uint8_t b, g, r, a; };  // ARGB 顺序
            uint32_t c;
        };
    };
}
```

转换函数：

```cpp
// 32 位颜色 → 128 位颜色
XMVECTOR XM_CALLCONV PackedVector::XMLoadColor(const XMCOLOR* pSource);

// 128 位颜色 → 32 位颜色
void XM_CALLCONV PackedVector::XMStoreColor(XMCOLOR* pDestination, FXMVECTOR V);
```

## 5.4 渲染管线总览

给定一个 3D 场景的几何描述和一个已定位的虚拟相机，**渲染管线（Rendering Pipeline）**是指生成 2D 图像所需的完整步骤序列。GPU 的渲染管线包含以下阶段：

```
输入装配 (IA) → 顶点着色器 (VS) → [可选: 曲面细分阶段]
    → [可选: 几何着色器 (GS)] → 裁剪 → 光栅化 (RS)
    → 像素着色器 (PS) → 输出合并 (OM)
```

| 阶段 | 类型 | 功能 |
|------|------|------|
| **输入装配 (IA)** | 固定功能 | 从内存读取顶点和索引，组装成几何图元 |
| **顶点着色器 (VS)** | 可编程 | 对每个顶点进行处理：坐标变换、动画、位移映射等 |
| **外壳着色器 (HS)** | 可编程 | 曲面细分：决定每个面片的细分等级 |
| **细分器 (TS)** | 固定功能 | 根据细分因子在面片内部插值生成新顶点 |
| **域着色器 (DS)** | 可编程 | 计算细分后顶点的最终属性 |
| **几何着色器 (GS)** | 可编程 | 对完整图元进行操作：可生成/销毁图元 |
| **裁剪** | 固定功能 | 丢弃视锥体外的几何，裁剪跨边界的多边形 |
| **光栅化 (RS)** | 固定功能 | 将三角形转换为像素片段，进行属性插值 |
| **像素着色器 (PS)** | 可编程 | 对每个像素片段计算最终颜色 |
| **输出合并 (OM)** | 固定功能 | 深度/模板测试、混合，写入后台缓冲区 |

管线中大部分阶段的数据流向是单向的：前一阶段的输出作为后一阶段的输入。只有输出合并阶段与 GPU 内存资源有双向交互（读取深度缓冲、写入颜色缓冲）。

## 5.5 输入装配阶段

输入装配器（Input Assembler, IA）从内存中读取几何数据（顶点和索引），将它们组装成几何图元（三角形、线段、点）。

### 5.5.1 顶点

在 Direct3D 中，顶点的概念比一般几何学中的"顶点"更广泛。除了位置信息，顶点还可以包含法线、纹理坐标、颜色、切线等任意属性。Direct3D 允许开发者**自定义顶点格式**，通过 `D3D12_INPUT_ELEMENT_DESC` 数组来描述顶点结构体的各个字段。

### 5.5.2 图元拓扑

顶点缓冲区只存储顶点列表，需要通过**图元拓扑（Primitive Topology）**告诉 GPU 如何把这些顶点组合成图元。

```cpp
void ID3D12GraphicsCommandList::IASetPrimitiveTopology(
    D3D_PRIMITIVE_TOPOLOGY Topology   // [in] 图元拓扑类型
);
```

#### `D3D_PRIMITIVE_TOPOLOGY` 详解

```cpp
typedef enum D3D_PRIMITIVE_TOPOLOGY {
    D3D_PRIMITIVE_TOPOLOGY_UNDEFINED                    = 0,
    D3D_PRIMITIVE_TOPOLOGY_POINTLIST                    = 1,   // 点列
    D3D_PRIMITIVE_TOPOLOGY_LINELIST                     = 2,   // 线列
    D3D_PRIMITIVE_TOPOLOGY_LINESTRIP                    = 3,   // 线带
    D3D_PRIMITIVE_TOPOLOGY_TRIANGLELIST                 = 4,   // 三角形列（最常用）
    D3D_PRIMITIVE_TOPOLOGY_TRIANGLESTRIP                = 5,   // 三角形带
    D3D_PRIMITIVE_TOPOLOGY_LINELIST_ADJ                 = 10,  // 线列（带邻接）
    D3D_PRIMITIVE_TOPOLOGY_LINESTRIP_ADJ                = 11,  // 线带（带邻接）
    D3D_PRIMITIVE_TOPOLOGY_TRIANGLELIST_ADJ             = 12,  // 三角形列（带邻接）
    D3D_PRIMITIVE_TOPOLOGY_TRIANGLESTRIP_ADJ            = 13,  // 三角形带（带邻接）
    D3D_PRIMITIVE_TOPOLOGY_1_CONTROL_POINT_PATCHLIST    = 33,  // 1 控制点面片
    // ... 2 到 32 控制点面片
    D3D_PRIMITIVE_TOPOLOGY_32_CONTROL_POINT_PATCHLIST   = 64,
} D3D_PRIMITIVE_TOPOLOGY;
```

| 拓扑类型 | 说明 | 顶点利用率 |
|---------|------|-----------|
| `POINTLIST` | 每个顶点绘制为一个独立的点。N 个顶点 → N 个点。 | — |
| `LINELIST` | 每两个顶点构成一条独立的线段。2N 个顶点 → N 条线。 | 无共享 |
| `LINESTRIP` | 顶点依次连接成线。N+1 个顶点 → N 条线。 | 内部顶点共享 |
| `TRIANGLELIST` | 每三个顶点构成一个独立的三角形。3N 个顶点 → N 个三角形。**最常用**。 | 无共享 |
| `TRIANGLESTRIP` | 顶点依次连接成三角形带。N 个顶点 → N-2 个三角形。偶数序号的三角形会自动调整顶点顺序以保持一致朝向。 | 高共享率 |

**三角形列（Triangle List）vs 三角形带（Triangle Strip）**：

- **三角形列**：灵活性高，三角形可以任意不连接。每个三角形独立定义，重复顶点需要通过索引来消除。
- **三角形带**：顶点共享率高，适合条带状几何（如地形、带状粒子效果）。但要求拓扑必须是连续的条带结构。

**带邻接信息的拓扑**：

`TRIANGLELIST_ADJ`、`TRIANGLESTRIP_ADJ` 等拓扑类型，每个三角形除了自身的 3 个顶点外，还附带 3 个相邻三角形的顶点（共 6 个顶点定义一个三角形）。这些邻接顶点只在几何着色器阶段可用，不会实际绘制。

**控制点面片拓扑**：

`N_CONTROL_POINT_PATCHLIST` 用于曲面细分阶段，每个面片由 N 个控制点定义。N 的取值范围是 1 到 32。

### 5.5.3 索引

当多个三角形共享顶点时，直接在顶点列表中重复顶点会造成内存浪费和顶点处理冗余。**索引（Index）** 解决了这个问题：

```
顶点列表（唯一顶点）        索引列表（每 3 个索引定义一个三角形）
v[0] = (x0, y0, z0)       { 0, 1, 2 }   → Triangle 0
v[1] = (x1, y1, z1)       { 0, 2, 3 }   → Triangle 1
v[2] = (x2, y2, z2)
v[3] = (x3, y3, z3)
```

- 顶点列表存储所有不重复的顶点数据。
- 索引列表存储整数索引，每三个索引指向顶点列表中的三个顶点，构成一个三角形。
- 索引通常用 16 位（`uint16_t`，最多 65536 个顶点）或 32 位（`uint32_t`）无符号整数存储。

**优势**：
1. 节省内存：索引（2-4 字节）比完整顶点结构（通常 24-48 字节）小得多。
2. 顶点复用：GPU 的**顶点缓存**可以复用已处理过的顶点，避免重复执行顶点着色器。

## 5.6 顶点着色器阶段

顶点着色器（Vertex Shader, VS）是 GPU 上执行的可编程阶段。它的核心任务可以概括为：**输入一个顶点，输出一个顶点**。GPU 会对每个绘制的顶点并行执行顶点着色器：

```cpp
// 概念上的执行模式
for (UINT i = 0; i < numVertices; ++i)
    outputVertex[i] = VertexShader(inputVertex[i]);
```

顶点着色器不仅可以访问输入顶点数据，还能访问 GPU 内存中的纹理、变换矩阵、光源等数据。常见用途包括坐标变换、顶点动画、位移映射等。

### 5.6.1 局部空间与世界空间

3D 模型通常在**局部空间（Local Space）**中建模——以物体中心为原点，坐标轴与物体对齐。这样建模最方便（如立方体的顶点很容易以原点为中心对称定义）。

要把物体放入场景，需要使用**世界变换（World Transform）**——通过**世界矩阵（World Matrix）**将顶点从局部空间转换到**世界空间（World Space）**。世界空间是场景中所有物体共享的全局坐标系。

世界矩阵通常由三个基本变换组合而成：`W = S × R × T`

- **S（缩放）**：调整物体在世界中的大小
- **R（旋转）**：调整物体的朝向
- **T（平移）**：调整物体的位置

```cpp
// DirectXMath 构建世界矩阵
XMMATRIX world = XMMatrixScaling(sx, sy, sz) *
                 XMMatrixRotationY(angle) *
                 XMMatrixTranslation(tx, ty, tz);
```

世界矩阵的行向量存储了局部坐标系的原点和三个坐标轴在世界空间中的齐次坐标。每个场景对象都有自己的世界矩阵。使用世界变换的好处：

1. 建模更方便（物体在原点、轴对齐）。
2. 模型可复用于不同场景。
3. 通过不同的世界矩阵，同一几何体可多次实例化绘制在不同位置。

### 5.6.2 视图空间

为了生成 2D 图像，需要在场景中放置一个**虚拟相机**。相机定义了观察者能看到的场景区域。与相机绑定的局部坐标系称为**视图空间（View Space）**或**相机空间（Camera Space）**：

- 相机位于原点 `(0, 0, 0)`
- 相机朝向 **+Z 轴** 方向（左手坐标系）
- **+X 轴** 指向相机右侧
- **+Y 轴** 指向相机上方

将顶点从世界空间转换到视图空间的变换称为**视图变换（View Transform）**，对应的矩阵是**视图矩阵（View Matrix）**。

视图矩阵是世界矩阵的逆。由于世界矩阵可分解为 `W = R × T`（旋转后平移），其逆矩阵有简洁形式：

```
V = W⁻¹ = T⁻¹ × R⁻¹ = T⁻¹ × Rᵀ
```

**构建视图矩阵的 Look-At 方法**：

给定相机位置 `Q`、观察目标 `T`、世界"上"方向 `j`（通常 `(0, 1, 0)`）：

1. 计算视线方向：`w = normalize(T - Q)`（相机局部 Z 轴，但指向 +Z，所以实际是反方向）
2. 计算相机右方向：`u = normalize(cross(j, w))`（相机局部 X 轴）
3. 计算相机上方向：`v = cross(w, u)`（相机局部 Y 轴）

```cpp
XMMATRIX XM_CALLCONV XMMatrixLookAtLH(
    FXMVECTOR EyePosition,    // [in] 相机位置 Q
    FXMVECTOR FocusPosition,  // [in] 观察目标 T
    FXMVECTOR UpDirection     // [in] 世界上方向 j
);
```

| 参数 | 说明 |
|------|------|
| `EyePosition` | 相机在世界空间中的位置。 |
| `FocusPosition` | 相机看向的世界空间中的点。 |
| `UpDirection` | 描述世界上方向的全局向量。通常设为 `(0, 1, 0)`。这个向量不需要精确垂直于视线，只需要大致指向上方，函数会重新正交化。 |

> **注意**：函数名中的 `LH` 表示**左手坐标系**（Left-Handed）。Direct3D 默认使用左手坐标系。如果使用右手坐标系，应改用 `XMMatrixLookAtRH`。

### 5.6.3 投影与齐次裁剪空间

视图空间中的相机还需要定义一个**视锥体（View Frustum）**——描述相机可见的空间区域。只有视锥体内的物体才会被渲染到屏幕上。

#### 定义视锥体

视锥体由以下四个参数定义：

| 参数 | 符号 | 说明 |
|------|------|------|
| 垂直视野角 | `fovY` | 相机在垂直方向上的视野角度（弧度） |
| 宽高比 | `aspect` | `width / height`，通常等于后台缓冲区的宽高比 |
| 近平面 | `n` | 视锥体近裁剪面到相机的距离（必须 > 0） |
| 远平面 | `f` | 视锥体远裁剪面到相机的距离 |

```
        top
    ┌─────────┐
   /│         │\
  / │    ●    │ \        ● = 相机（原点）
 /  │   /|\   │  \
/   │  / | \  │   \      w = 视锥体宽度
│   │ /  |  \ │   │      h = 视锥体高度
│   │/   |   \│   │      n = 近平面距离
│   └─────────┘   │      f = 远平面距离
│        n        │
│                 │
│        f        │
└─────────────────┘
```

**计算投影平面距离**：

要使垂直视野角为 `fovY`，投影平面（位于近平面处）到原点的距离 `d` 为：

```
d = 1 / tan(fovY / 2)
```

#### 投影变换

透视投影将 3D 顶点投影到 2D 投影平面上，使得：
- 平行线汇聚到消失点
- 物体大小随深度增加而减小

对于视图空间中的一点 `(x, y, z)`，其在投影平面 `z = d` 上的投影坐标为：

```
x' = x × d / z
y' = y × d / z
```

这个变换是**非线性**的（除以 z），无法直接用矩阵乘法表示。解决方法是利用**齐次坐标**：将除法推迟到后面的**透视除法**阶段。

#### 归一化设备坐标（NDC）

投影后的坐标需要进一步归一化，消除对宽高比的依赖：

- X 从 `[-r, r]` 映射到 `[-1, 1]`
- Y 从 `[-1, 1]` 保持不变
- Z 从 `[n, f]` 映射到 `[0, 1]`（深度缓冲需要）

NDC 空间中的坐标满足：

```
-1 ≤ x ≤ 1
-1 ≤ y ≤ 1
 0 ≤ z ≤ 1
```

#### 透视投影矩阵

DirectXMath 提供函数直接构建透视投影矩阵：

```cpp
XMMATRIX XM_CALLCONV XMMatrixPerspectiveFovLH(
    FLOAT FovAngleY,      // [in] 垂直视野角（弧度）
    FLOAT AspectRatio,    // [in] 宽高比
    FLOAT NearZ,          // [in] 近平面距离（> 0）
    FLOAT FarZ            // [in] 远平面距离
);
```

| 参数 | 说明 |
|------|------|
| `FovAngleY` | 垂直方向上的视野角度，以弧度为单位。人眼的垂直视野约为 60°（π/3 弧度）。常见游戏设置：60°~90°。 |
| `AspectRatio` | 视口宽高比，通常设为 `(float)width / height`。例如 1280×720 时，`aspect = 1280.0f / 720.0f ≈ 1.778`。 |
| `NearZ` | 近平面距离。设为太小（如 0.001）会导致深度缓冲精度问题（Z-fighting）。典型值：1.0 或 0.1。 |
| `FarZ` | 远平面距离。设为太大也会降低深度精度。根据场景大小调整，如 100.0、1000.0。 |

> **注意**：函数名中的 `LH` 表示左手坐标系。在左手系中，相机看向 +Z 方向，近平面为 `z = n`，远平面为 `z = f`。NDC 中 Z 的范围是 `[0, 1]`。如果使用右手坐标系，应改用 `XMMatrixPerspectiveFovRH`，此时 NDC 中 Z 的范围是 `[-1, 1]`（OpenGL 风格）。

**投影矩阵的数学形式（左手坐标系）**：

```
| aspect × d   0        0              0     |
|     0        d        0              0     |
|     0        0    f/(f-n)      -nf/(f-n)  |
|     0        0        1              0     |
```

其中 `d = 1 / tan(fovY / 2)`。

**完整的顶点坐标变换链**：

```
局部空间坐标  ──[世界矩阵 W]──→  世界空间坐标
世界空间坐标  ──[视图矩阵 V]──→  视图空间坐标
视图空间坐标  ──[投影矩阵 P]──→  齐次裁剪空间坐标 (x, y, z, w)
齐次裁剪坐标  ──[透视除法 /w]──→  NDC 坐标 (x/w, y/w, z/w)
NDC 坐标      ──[视口变换]────→  屏幕像素坐标
```

在顶点着色器中，通常将 W、V、P 三个矩阵预先相乘为**组合变换矩阵**：

```cpp
XMMATRIX worldViewProj = world * view * proj;
```

顶点着色器输出的是**齐次裁剪空间坐标**（`SV_Position` 语义），GPU 后续会自动完成透视除法和视口变换。

---

## 5.7 曲面细分阶段

曲面细分（Tessellation）是可选的可编程阶段，位于顶点着色器之后、几何着色器之前。它包含三个子阶段：

1. **外壳着色器（Hull Shader）**：输出每个面片的细分因子（Tessellation Factor），决定面片被细分多少。
2. **细分器（Tessellator）**：固定功能阶段，根据细分因子在参数域内生成新的顶点位置（UV 坐标）。
3. **域着色器（Domain Shader）**：接收细分器生成的参数坐标，计算每个新顶点的最终 3D 位置和其他属性。

曲面细分的动机：在低多边形模型基础上动态增加三角形数量，提升细节。例如远处的地形使用低细分，近处使用高细分（基于距离的 LOD）。

详细内容见第 14 章。

---

## 5.8 几何着色器阶段

几何着色器（Geometry Shader, GS）是可选的可编程阶段，输入一个完整图元（点、线或三角形），输出零个或多个新图元。

**几何着色器的独特能力**：
- 可以创建新图元（如将点扩展为四边形公告板）。
- 可以销毁图元（输出零个顶点）。
- 可以修改图元类型（如输入三角形，输出线段）。

典型应用：公告板（Billboard）、粒子系统、 fur 渲染、轮廓线提取。

详细内容见第 12 章。

## 5.9 裁剪

完全在视锥体外的几何需要被丢弃，与视锥体边界相交的几何需要被**裁剪（Clip）**，只保留内部部分。

视锥体由六个平面界定：上、下、左、右、近、远。裁剪依次对每个平面进行：保留平面正半空间内的部分，丢弃负半空间。

GPU 自动执行裁剪，开发者无需手动实现。但了解裁剪机制很重要：

- 裁剪后的多边形可能增加顶点数（一个三角形被裁剪后可能变成四边形或五边形）。
- 硬件会自动将裁剪后的凸多边形三角化。
- 裁剪在**4D 齐次空间**中进行，在透视除法之前完成。这是为了避免透视除法后出现 `w = 0` 的退化情况。

**齐次裁剪空间的边界**：

在透视除法之前，4D 点 `(x, y, z, w)` 如果在视锥体内，满足：

```
-w ≤ x ≤ w
-w ≤ y ≤ w
 0 ≤ z ≤ w   (左手坐标系)
```

这些边界对应六个裁剪平面：左 `w = -x`、右 `w = x`、下 `w = -y`、上 `w = y`、近 `z = 0`、远 `z = w`。

## 5.10 光栅化阶段

光栅化（Rasterization）的核心任务是从投影后的 3D 三角形计算出覆盖的像素颜色。

### 5.10.1 视口变换

裁剪完成后，硬件执行**透视除法**（除以 `w`），从齐次裁剪空间转换到 NDC 空间。然后 NDC 的 XY 坐标通过**视口变换**映射到后台缓冲区的像素坐标：

```
屏幕 X = (NDC_X + 1) × 0.5 × ViewportWidth  + ViewportTopLeftX
屏幕 Y = (1 - NDC_Y) × 0.5 × ViewportHeight + ViewportTopLeftY
```

> Y 坐标取反是因为 NDC 的 +Y 向上，而屏幕坐标的 +Y 向下。

`D3D12_VIEWPORT` 结构中的 `MinDepth` 和 `MaxDepth` 控制 Z 值到深度缓冲的映射。默认 `[0, 1]` 表示完整深度范围。

### 5.10.2 背面剔除

三角形有两个面。通过**顶点环绕顺序（Winding Order）**来区分前后：

- 按照顶点顺序 `v0 → v1 → v2` 计算法向量 `n = normalize(cross(v1 - v0, v2 - v0))`。
- 法向量指向的面是**正面（Front Face）**，另一面是**背面（Back Face）**。
- Direct3D 默认约定：**顺时针**环绕的三角形（从相机视角看）是正面，**逆时针**是背面。

对于封闭的实心物体，背面三角形永远被正面三角形遮挡，因此可以安全剔除。**背面剔除（Backface Culling）**可以将近一半的三角形从管线中剔除，显著提升性能。

剔除模式可通过光栅器状态设置（`D3D12_RASTERIZER_DESC` 的 `CullMode` 字段）：

| 模式 | 说明 |
|------|------|
| `D3D12_CULL_MODE_NONE` | 不剔除，双面都绘制 |
| `D3D12_CULL_MODE_BACK` | 剔除背面（默认） |
| `D3D12_CULL_MODE_FRONT` | 剔除正面 |

### 5.10.3 顶点属性插值

光栅化确定了一个三角形覆盖了哪些像素后，需要为每个被覆盖的像素计算**顶点属性的插值**——包括颜色、法线、纹理坐标、深度值等。

插值在**屏幕空间**进行，但必须保持**3D 空间中的线性关系**（称为**透视校正插值**）。原因：透视投影后，屏幕空间中的均匀步长对应 3D 空间中的非均匀步长。

```
3D 空间中的线段：          屏幕空间中的投影：
    A ●──────────● B         A' ●────●────● B'
       均匀步长                  非均匀步长
```

GPU 硬件自动执行透视校正插值，开发者无需关心数学细节。

---

## 5.11 像素着色器阶段

像素着色器（Pixel Shader, PS）对光栅化生成的每个**像素片段（Pixel Fragment）**执行。输入是插值后的顶点属性，输出是一个颜色值。

像素着色器可以实现：
- 简单的常量颜色输出
- 纹理采样
- 逐像素光照（Phong/Blinn-Phong）
- 法线贴图
- 阴影计算
- 反射/折射效果

```cpp
// HLSL 像素着色器示例
float4 PS(VertexOut pin) : SV_Target
{
    return float4(1.0f, 0.0f, 0.0f, 1.0f);  // 纯红色
}
```

`SV_Target` 语义表示输出的是渲染目标颜色。

---

## 5.12 输出合并阶段

像素着色器输出的像素片段进入**输出合并（Output Merger, OM）**阶段，进行以下处理：

1. **深度测试（Depth Test）**：比较像素片段的深度值与深度缓冲中对应位置的值。如果新片段更远（深度值更大），则丢弃该片段。
2. **模板测试（Stencil Test）**：根据模板缓冲中的值和设置的比较函数决定是否丢弃像素。
3. **混合（Blending）**：将新像素颜色与后台缓冲区中已有颜色按指定公式混合，用于实现透明、加法混合等效果。
4. **写入**：通过所有测试的像素被写入后台缓冲区（和深度/模板缓冲）。

```cpp
void ID3D12GraphicsCommandList::OMSetRenderTargets(
    UINT NumRenderTargetDescriptors,
    const D3D12_CPU_DESCRIPTOR_HANDLE* pRenderTargetDescriptors,
    BOOL RTsSingleHandleToDescriptorRange,
    const D3D12_CPU_DESCRIPTOR_HANDLE* pDepthStencilDescriptor
);
```

输出合并阶段是管线的最后一个阶段，负责将最终的像素数据写入 GPU 资源。混合和深度/模板测试的具体配置将在后续章节深入讲解。
